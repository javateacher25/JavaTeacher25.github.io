    <div class="container">
        <h1>Development Fundamentals</h1>
        <p class="lead">Building the foundation for your journey as a software developer. Understanding the fundamentals is crucial before diving into any programming language or framework.</p>

        <div class="topic-section">
            <h2><i class="bi bi-clock-history"></i> Computer History</h2>
            <p>Understanding the evolution of computers helps us appreciate modern programming and make better architectural decisions.</p>
            
            <div class="topic-card">
                <h3>Why This Matters</h3>
                <p>Learning computer history isn't just about memorizing dates and names. It helps you understand:</p>
                <ul>
                    <li><strong>Evolution of Programming Paradigms:</strong> From machine code to high-level languages, see how abstraction evolved</li>
                    <li><strong>Hardware-Software Relationship:</strong> Why certain optimizations matter and how software interacts with hardware</li>
                    <li><strong>Problem-Solving Approaches:</strong> Learn from the challenges early pioneers faced and their elegant solutions</li>
                    <li><strong>Industry Trends:</strong> Recognize patterns in technology cycles to better predict future directions</li>
                </ul>
                
                <h3>What We'll Cover</h3>
                <ul>
                    <li>From ENIAC to modern computing</li>
                    <li>The birth of programming languages</li>
                    <li>Evolution of operating systems</li>
                    <li>The internet revolution and its impact on software development</li>
                    <li>Cloud computing and distributed systems era</li>
                </ul>
            </div>
        </div>

        <div class="topic-section">
            <h2><i class="bi bi-cpu"></i> Digital Circuits</h2>
            <p>Understanding how computers work at the fundamental level - from electricity to executable code.</p>
            
            <div class="topic-card">
                <h3>Why This Matters</h3>
                <p>As a backend developer, knowing digital circuits gives you:</p>
                <ul>
                    <li><strong>Deep Understanding:</strong> Know what happens when you write <code>int x = 5;</code> at the hardware level</li>
                    <li><strong>Performance Optimization:</strong> Make informed decisions about data types, memory usage, and CPU operations</li>
                    <li><strong>Debugging Skills:</strong> Understand low-level errors and memory-related issues better</li>
                    <li><strong>Architecture Awareness:</strong> Appreciate why certain operations are faster than others</li>
                    <li><strong>Bit Manipulation:</strong> Master bitwise operations used in cryptography, compression, and system programming</li>
                </ul>
                
                <h3>What We'll Cover</h3>
                <ul>
                    <li>Binary number systems and Boolean algebra</li>
                    <li>Logic gates and circuit design</li>
                    <li>Memory: RAM, ROM, Cache hierarchy</li>
                    <li>CPU architecture and instruction cycles</li>
                    <li>How high-level code becomes machine instructions</li>
                    <li>Data representation: integers, floating-point, characters</li>
                </ul>
            </div>
        </div>

        <div class="topic-section">
            <h2><i class="bi bi-diagram-3"></i> Algorithms and Data Structures</h2>
            <p>The heart of computer science - learning to solve problems efficiently and elegantly.</p>
            
            <div class="topic-card">
                <h3>Why This Matters</h3>
                <p>This is arguably the most important topic for any developer:</p>
                <ul>
                    <li><strong>Problem-Solving Foundation:</strong> Learn systematic approaches to break down complex problems</li>
                    <li><strong>Performance:</strong> Choose the right data structure and your app can be 100x faster</li>
                    <li><strong>Code Quality:</strong> Write cleaner, more maintainable code that scales</li>
                    <li><strong>Technical Interviews:</strong> Essential for passing coding interviews at top companies</li>
                    <li><strong>Real-World Applications:</strong> From database indexing to search engines, algorithms power everything</li>
                    <li><strong>Computational Thinking:</strong> Develop the mindset to approach any technical challenge</li>
                </ul>
                
                <h3>What We'll Cover</h3>
                
                <h4>Data Structures:</h4>
                <ul>
                    <li>Arrays and Dynamic Arrays</li>
                    <li>Linked Lists (Single, Double, Circular)</li>
                    <li>Stacks and Queues</li>
                    <li>Hash Tables and Hash Maps</li>
                    <li>Trees (Binary Trees, BST, AVL, Red-Black)</li>
                    <li>Heaps and Priority Queues</li>
                    <li>Graphs and their representations</li>
                </ul>
                
                <h4>Algorithms:</h4>
                <ul>
                    <li>Searching: Linear Search, Binary Search</li>
                    <li>Sorting: Bubble, Selection, Insertion, Merge, Quick, Heap Sort</li>
                    <li>Recursion and Backtracking</li>
                    <li>Dynamic Programming</li>
                    <li>Greedy Algorithms</li>
                    <li>Graph Algorithms: DFS, BFS, Dijkstra's, A*</li>
                    <li>String Algorithms and Pattern Matching</li>
                </ul>
                
                <h4>Complexity Analysis:</h4>
                <ul>
                    <li>Big O Notation</li>
                    <li>Time and Space Complexity</li>
                    <li>Best, Average, and Worst Case Analysis</li>
                </ul>
            </div>
        </div>

        <div class="topic-section">
            <h2><i class="bi bi-git"></i> Git & Version Control</h2>
            <p>The essential tool for modern software development - collaborate, track changes, and never lose your work.</p>
            
            <div class="topic-card">
                <h3>Why This Matters</h3>
                <p>Git is not optional in modern development - it's mandatory. Here's why:</p>
                <ul>
                    <li><strong>Collaboration:</strong> Work with teams without stepping on each other's toes</li>
                    <li><strong>History & Rollback:</strong> Track every change and revert mistakes instantly</li>
                    <li><strong>Branching Strategy:</strong> Work on features independently, experiment safely</li>
                    <li><strong>Code Review:</strong> Professional teams review all code before merging</li>
                    <li><strong>Deployment:</strong> Git integrates with CI/CD pipelines for automated deployments</li>
                    <li><strong>Portfolio:</strong> GitHub is your resume - employers check your repositories</li>
                    <li><strong>Open Source:</strong> Contribute to projects used by millions worldwide</li>
                </ul>
                
                <h3>What We'll Cover</h3>
                
                <h4>Git Fundamentals:</h4>
                <ul>
                    <li>Understanding version control systems</li>
                    <li>Repository initialization and cloning</li>
                    <li>Staging and committing changes</li>
                    <li>Working directory, staging area, and repository</li>
                    <li>Git workflow and best practices</li>
                </ul>
                
                <h4>Branching & Merging:</h4>
                <ul>
                    <li>Creating and switching branches</li>
                    <li>Merge strategies and conflict resolution</li>
                    <li>Rebasing vs Merging</li>
                    <li>Git Flow and branching models</li>
                    <li>Feature branches, hotfixes, and releases</li>
                </ul>
                
                <h4>Collaboration:</h4>
                <ul>
                    <li>Remote repositories (GitHub, GitLab, Bitbucket)</li>
                    <li>Pushing and pulling changes</li>
                    <li>Pull requests and code reviews</li>
                    <li>Forking and contributing to open source</li>
                    <li>Managing merge conflicts in teams</li>
                </ul>
                
                <h4>Advanced Topics:</h4>
                <ul>
                    <li>Cherry-picking commits</li>
                    <li>Stashing changes</li>
                    <li>Interactive rebase</li>
                    <li>Git hooks and automation</li>
                    <li>.gitignore and repository management</li>
                    <li>Git tags and releases</li>
                </ul>
            </div>
        </div>

        <div class="learning-path">
            <h2><i class="bi bi-signpost-2"></i> Our Approach</h2>
            <p>In this module, we focus on <strong>understanding the basics</strong> rather than mastering everything. Our goal is to give you:</p>
            <ul>
                <li><strong>How Computers Work</strong> - The fundamental concepts behind computing</li>
                <li><strong>What Programming Means</strong> - Understanding the nature of software development</li>
                <li><strong>How Algorithms Are Made</strong> - The thinking process behind problem-solving</li>
                <li><strong>Essential Tools</strong> - Git for managing your code from day one</li>
            </ul>
            <p>We're building your <em>conceptual foundation</em> - you'll dive deeper into each area as you progress through Java SE, Jakarta EE, and Spring.</p>
        </div>
